API CRUD

On va ici appliquer les principes d'une architecture REST.

CORS filter

Ajoutez un filtre CORS : à votre descripteur de déploiement (web.xml) :

    <filter>
        <filter-name>CorsFilter</filter-name>
        <filter-class>org.apache.catalina.filters.CorsFilter</filter-class>
        <init-param>
          <param-name>cors.allowed.origins</param-name>
          <param-value>https://editor.swagger.io</param-value>
        </init-param>
        <init-param>
          <param-name>cors.allowed.methods</param-name>
          <param-value>GET,POST,PUT,DELETE,OPTIONS</param-value>
        </init-param>
        <init-param>
          <param-name>cors.support.credentials</param-name>
          <param-value>true</param-value>
        </init-param>
        <init-param>
          <param-name>cors.allowed.headers</param-name>
          <param-value>Authorization</param-value>
        </init-param>
  </filter>
  <filter-mapping>
    <filter-name>CorsFilter</filter-name>
    <url-pattern>/*</url-pattern>
  </filter-mapping>

Contrôleurs

  • Pour chaque ressource :
    • Créez un package 'controller' dans le package 'application'.
    • Créez une classe 'Controller' qui étends 'HttpServlet' et surcharge les méthodes 'doGet', 'doPost', 'doPut' et 'doDelete'.

      Note : Pour chaque servlet, vous utiliserez l'annotation @WebServlet pour définir son nom et son mapping. Vous pouvez cependant faire la même chose en y déclarant dans le descripteur de déploiement (web.xml).

DTO

  • Pour chaque ressource, dans le package 'application' :

    • Créez un package 'dto', puis dans ce dernier créez:
      • un POJO 'InDto' pour la récupération des corps de requêtes, correspondant au domain mais sans son identifiant technique.
      • un POJO 'OutDto' pour le renvoie des données dans la réponse, correspondant au domain avec son identifiant technique mais sans les données sensibles (par exemple : mot de passe).
    • Créez un package 'mapper', puis créez un mapper MapStruct 'DtoMapper' :

    ```

    <nom_resource>DtoMapper INSTANCE = Mappers.getMapper(<nom_resource>DtoMapper.class);
<nom_resource> inDtoToDomain(In<nom_resource>Dto in);
Out<nom_resource>Dto domainToOutDto(<nom_resource> domain);
List<Out<nom_resource>Dto> domainListToOutDto(List<<nom_resource>> domainList);

    ```

Front Controller

  • Dans le package 'gateway', créez le package 'controller', puis :
    • Créez une classe 'DispatcherServlet' qui étends 'HttpServlet' :
      • Le front controller doit être le seul à capter les URLs, cette servlet doit donc être chargée en première et on donc spécifier le paramètre "loadOnStartup" à 1.
      • Surchargez les méthodes 'doGet', 'doPost', 'doPut' et 'doDelete'
      • Créez une méthode privée 'dispatchRequest(final HttpServletRequest request, final HttpServletResponse response)' qui sera utilisé dans chaque méthodes 'do'
        • Chaque requête visera une ressource, et donc l'URI de la requête contiendra le nom de la ressource visée (task, user ou board):

          - Récupérez l'URI de la requête avec request.getRequestURI() puis faites en sortes de récupérer la partie de l'URI qui vous intéresse.
