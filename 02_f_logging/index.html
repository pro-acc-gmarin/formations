Logging

Slf4j

Maven

Ajouter les dépendances suivantes dans votre 'pom.xml' :

    <dependency>
        <groupId>org.apache.logging.log4j</groupId>
        <artifactId>log4j-api</artifactId>
        <version>2.20.0</version>
    </dependency>
    <dependency>
        <groupId>org.apache.logging.log4j</groupId>
        <artifactId>log4j-core</artifactId>
        <version>2.20.0</version>
    </dependency>

Ajouter dans le build la définition du répertoire de resource:

    <resources>
      <resource>
        <directory>src/main/resources</directory>
        <filtering>true</filtering>
      </resource>
    </resources>

Configuration

  • Créer un dossier 'configuration' dans votre dossier de resources, puis créer un fichier 'log4j2.xml' dedans.
  • Dans le package 'gateway' puis 'configuration', créer une classe 'Log4jConfiguration' qui implémentera une méthode statique 'setConfigLocation' qui chargera le fichier de configuration log4j2 :
     public static void setConfigLocation(){
        LoggerContext context = (LoggerContext) LogManager.getContext(false);
        try {
            context.setConfigLocation(Log4j2Configuration.class.getResource("/configuration/log4j2.xml").toURI());
        } catch (final URISyntaxException exception) {
            throw new RuntimeException(exception);
        }
    }

  • Appeler cette méthode statique, dans la méthode 'contexteInitialized' de votre 'InitServletContext'.

  • Dans le fichier log4j2.xml, faites en sorte de déclarer un appender pour chaque resource et un autre pour tout ce qui est application en général:

    • Chaque appender doit écrire dans un fichier horodaté différent.
    • Définir le logger qui utilisera ces appenders en level debug.

Utilisation

  • AOP : Logger les exceptions remontées dans l'aspect d'Handling Exception.
