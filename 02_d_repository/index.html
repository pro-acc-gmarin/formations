Repository

Entity

Pour chaque ressource, dans le package 'infrastructure' : - Créer un package entity, puis dans ce package créer un POJO 'Entity' avec les mêmes propriétés que le domain sans les identifiants techniques. - Créer un package mapper, puis dans ce package créer une classe 'Mapper' : <nom_resource>Mapper INSTANCE = Mappers.getMapper(<nom_resource>Mapper.class); <nom_resource> entityToDomain(final <nom_resource>Entity entity); <nom_resource>Entity domainToEntity(final <nom_resource> domain); List<<nom_resource>> entityListToDomainList(final List<<nom_resource>Entity> entityList);

Design Pattern : DAO

Pour chaque ressource, dans le package 'infrastructure' : - Créer un package 'spi' puis dedans, créer une interface 'DaoSpi' : <nom_resource>Entity add(final <nom_resource>Entity object); void delete(final String id); Optional<<nom_resource>Entity> update(final <nom_resource>Entity object, final String id); List<<nom_resource>Entity> getAll(); Optional<<nom_resource>Entity> getById(final String id); - Créer un package 'dao' puis dedans , créer une classe 'Dao' qui implémente la spi précédemment créée.

Intérargir avec la base de donnée

- Récupérez une connection et faire une requête préparée avec la méthode **'prepareStatement(...)**' de votre connection, à laquelle vous passerez votre requête SQL paramétrée :
    - ex pour DELETE :

    ```
        try (Connection connection = ConnectionPool.getInstance().getConnection()) {
            final PreparedStatement statement = connection.prepareStatement("DELETE FROM ressource WHERE id = ?", Statement.RETURN_GENERATED_KEYS);
            statement.setString(1, id);
            if (statement.executeUpdate() == 0) {
                throw new SQLException("Deleting board failed, no rows affected.");
            }
        }
    ```

Requête SQL par annotation

- Dans votre package **'utils'** à la raçine, créer un package **'annotation'** puis créer l'annotation **'@SQLQuery'**:

```
    @Retention(RetentionPolicy.RUNTIME)
    @Target(ElementType.METHOD)
    public @interface PreparedQuery {
        String value();
    }
```

- Afin de récupérer la valeur passée dans l'annotation, on va utiliser la réflection java :
    - On récupère le nom de la méthode courante :

    ```
    final String currentMethodName = Thread.currentThread().getStackTrace()[1].getMethodName();
    ```

    - On récupère la méthode courante :
    ```
    final Method currentMethod = enclosingClass.getDeclaredMethod(methodName);
    ```

    - On récupère l'annotation :
    ```
    final SQLQuery annotation = currentMethod.getAnnotation(SQLQuery.class);
    ```

    - Puis sa valeur avec **'value()'** (vous pouvez faire une classe Helper pour la récupération de sa valeur):

    ```
        @PreparedQuery("DELETE FROM resource WHERE id = ?")
        public void delete(String id) {
            String currentMethodName = Thread.currentThread().getStackTrace()[1].getMethodName();
            final Method currentMethod = enclosingClass.getDeclaredMethod(methodName);
            final SQLQuery annotation = currentMethod.getAnnotation(SQLQuery.class);
            try (Connection connection = ConnectionPool.getInstance().getConnection()) {
                PreparedStatement statement = connection.prepareStatement(annotation.value(), Statement.RETURN_GENERATED_KEYS);
                statement.setString(1, id);
                int affectedRows = statement.executeUpdate();
                if (affectedRows == 0) {
                    throw new SQLException("Deleting user failed, no rows affected.");
                }
            } catch (NoSuchMethodException | SQLException e) {
                throw new RuntimeException(e);
            }
        }
    ```
