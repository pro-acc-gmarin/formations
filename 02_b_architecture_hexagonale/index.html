Architecture : Monolithe hexagonal

Créez l'arborescence de package suivante pour votre application :

/
    board
        application
        domain
            api
            data
            spi
        infrastructure
    user
        application
        domain
            api
            data
            spi
        infrastructure
    task
        application
        domain
            api
            data
            spi
        infrastructure
    gateway
        configuration (package déjà existant à déplacer)

Domain & Ports

  • Pour chaque ressource :

    • Définissez les POJOs (objet domain) représentant les données définies en base (user, task et board) dans le package 'data' du package 'domain'.

    • Définissez dans le package 'api' une interface 'ServicePort' avec les signatures de méthode suivantes :

      ```

      Board add(final Board board);
void delete(final String id);
Board update(final Board board, final String id);
List<Board> getAll();
Board getById(final String id);

      ```

    • Définissez dans le package 'spi' une interface 'PersistencePort' avec les mêmes signatures de méthode.

    • Définissez dans le package 'api' l'implémentation 'ServiceImpl' de votre interface du 'service port'.

      • Cette implémentation va utiliser le persistence port, qui sera passé en propriété de classe :

      ```

      private final <nom_ressource>PersistencePort <nom_ressource>PersistencePort;

public <nom_ressource>ServiceImpl(<nom_ressource>PersistencePort <nom_ressource>PersistencePort) {
    this.<nom_ressource>PersistencePort = <nom_ressource>PersistencePort;
}

      ``` - L'implémentation des méthodes va donc appeler les méthodes correspondantes du persistence port, ex :

      public <nom_ressource> add(<nom_ressource> <nom_ressource>) { return this.<nom_ressource>PersistencePort.add(<nom_ressource>); }
