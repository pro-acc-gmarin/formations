Aspect Oriented Programming

Maven

Ajouter les dépendances suivantes dans votre 'pom.xml' :

<dependency>
    <groupId>org.aspectj</groupId>
    <artifactId>aspectjweaver</artifactId>
    <version>1.9.7</version>
</dependency>
<dependency>
    <groupId>org.aspectj</groupId>
    <artifactId>aspectjrt</artifactId>
    <version>1.9.7</version>
</dependency>

Ajouter le plugin :

    <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>aspectj-maven-plugin</artifactId>
        <version>${aspectj.plugin.version}</version>
        <configuration>
            <source>${compiler.version}</source>
            <target>${compiler.version}</target>
            <complianceLevel>${compiler.version}</complianceLevel>
            <showWeaveInfo>true</showWeaveInfo>
            <sources>
                <source>
                    <basedir>src/main/java</basedir>
                    <includes>
                        <include>utils/aspect/*.java</include>
                        <include>board/application/controller/*.java</include>
                        <include>user/application/controller/*.java</include>
                        <include>task/application/controller/*.java</include>
                        <include>board/infrastructure/dao/*.java</include>
                        <include>user/infrastructure/dao/*.java</include>
                        <include>task/infrastructure/dao/*.java</include>
                    </includes>
                </source>
            </sources>
        </configuration>
        <dependencies>
            <dependency>
                <groupId>org.aspectj</groupId>
                <artifactId>aspectjtools</artifactId>
                <version>${aspectj.version}</version>
            </dependency>
            <dependency>
                <groupId>org.aspectj</groupId>
                <artifactId>aspectjweaver</artifactId>
                <version>${aspectj.version}</version>
            </dependency>
        </dependencies>
        <executions>
            <execution>
                <goals>
                    <goal>compile</goal>
                </goals>
            </execution>
        </executions>
    </plugin>

Exception Handling

  • Dans votre package 'annotation', créer une annotation '@HandleException' au RUNTIME et appliquable sur des méthodes.
    • Dans chaque controller de ressources, annotée la méthode dispatchAction avec cette nouvelle annotation.
  • Dans votre package 'utils', créer une classe 'HandlingExceptionAspect', on utilisera ici les annotations d'AspectJ afin de définir notre aspect, ses pointcuts et advices:
    • Annoter la classe avec @Aspect
    • Créer les méthodes vides annotées @Pointcut ciblant les méthodes 'dispatchAction' de chaque controller de ressource.
      • Le pointcut doit cibler l'execution de la méthode dispatchAction et que celle-ci ait l'annotation @HandleException.
    • Créer les advices annotés '@AfterThrowing' pour capturer les exceptions remontées par la cible de votre pointcut passé en paramètre.
      • Dans le code de vos advices, récupérer l'argument du joinPoint correspondant à votre HttpServletResponse, afin de pouvoir selon l'exception remontée, mettre le HTTP status adapté.
